Pricoin (PRIC) is a Bitcoin Core fork where every non-coinbase transaction is confidential by consensus. Sender, recipient, and amount are hidden by construction — there is no transparent send path to fall back to.

What is Pricoin?

Pricoin is a peer-to-peer electronic cash system that takes the Bitcoin Core codebase and replaces transparent transactions with a privacy-mandatory transaction format inspired by Monero's RingCT and CryptoNote stealth addresses. Three primitives, all required:

∑

Confidential amounts

Every output stores a Pedersen commitment C = v·H + b·G instead of a cleartext value. A Borromean rangeproof asserts 0 ≤ v < 2⁶⁴ without revealing v.

⌖

Stealth addresses

Your reusable identity is a pair of keys (A, B). Each payment derives a fresh one-time output script via ECDH. Two payments to the same identity look uncorrelated on chain.

○

Ring signatures

Each spend hides among N = 16 ring candidates via multi-layer CLSAG. Double-spends are caught by a tx-invariant key image, not by linking inputs.

Status: experimental / educational. This is a toy implementation of a real privacy stack. Bugs in confidential transactions or ring signatures break privacy silently without breaking consensus. Do not use this code with real money.

How Pricoin is different

Pricoin keeps Bitcoin's economics — 50 PRIC initial reward, ~21 million cap, halving every ~4 years — but throws out everything about transparent transactions. The chain looks Bitcoin-shaped from the outside (UTXOs, blocks, scripts) but every spend has Pedersen amounts, ring inputs, and one-time stealth outputs.

Pricoin vs Monero

Pricoin is essentially a Monero-style privacy stack on Bitcoin's curve and codebase. The cryptographic primitives are nearly the same — what differs is the platform.

Honest gap: Pricoin's rangeproofs are about 2× the size of Monero's because Pricoin still uses Borromean — Bulletproofs proper aren't shipped in libsecp256k1-zkp yet. Monero's RingCT has also been hardened by years of attacks (EAE on temporal decoys, Janus, etc.); Pricoin has been hardened by zero.

Pricoin vs Zcash

Pricoin and Zcash both hide value, sender, and recipient — but the underlying math is fundamentally different, and so are the engineering trade-offs.

Anonymity set per spendRing size 16 (16 candidates)Entire shielded pool (millions of notes)

Verification costFast (no pairings)Pairing-heavy (slower, but still sub-second)

In Zcash's shielded pool, your spend is hidden among every shielded note that has ever existed — the anonymity set is gigantic, and decoy selection is irrelevant. The cost is heavy cryptography (zk-SNARKs), historical trusted setups, and the practical fact that most Zcash usage today is still on transparent addresses.

In Pricoin (and Monero) every spend is private by construction, but each one only hides among 16 candidates. Bad decoy selection or temporal heuristics have repeatedly leaked information for specific transactions. The crypto is simpler and faster, the privacy is broader (everyone is private) but shallower (small ring).

Pricoin vs Particl

Particl is the closest existing project by lineage and cryptography — it's also a Bitcoin Core fork that vendored Monero-style ring confidential transactions. The difference is one of policy: Particl lets each transaction choose its own privacy level; Pricoin does not give the user that choice.

Why mandatory beats optional

Optional privacy creates an anonymity-set problem. If most users send transparently, the few who opt into private transactions stand out — and the on-chain signal of "this user chose privacy" is itself information. Worse, the anonymity set for ring sigs is drawn from past private outputs only, so it grows slowly when most of the chain is transparent.

This is exactly the argument Monero made when it removed its transparent-tx option in 2017: the only way to get real privacy from ring signatures is to put everyone in the same anonymity set, by default, with no escape hatch. Pricoin takes the same position at the consensus layer — there is no v1/v2/v3 send path to fall back to, so there is no "the people who chose privacy" cohort to single out.

Particl's flexibility is genuinely useful for some use cases (regulatory reporting, exchange deposits, etc.). The trade-off is that the privacy guarantee for individual users is weaker than what a mandatory-privacy chain provides.

Coinomics

Pricoin keeps Bitcoin's per-block reward (50 PRIC, halving every ~4 years) but runs at 4× the block rate (150 s vs 600 s). Holding the halving cadence at 4 years means each era spans 4× as many blocks as Bitcoin's, and each era therefore emits 4× as many coins. The supply cap lands at ~84 million PRIC, not 21 million.

Coinbase remains transparent — subsidy is auditable, total emission verifiable in cleartext, and miners' rewards visible. After coinbase maturity, the funder uses walletsendct to roll mined transparent coins into a confidential output. From that point on, every spend is private.

Mining

Pricoin uses RandomX for proof-of-work — the same CPU-friendly, ASIC-resistant algorithm Monero uses. The block identity hash stays SHA-256d (so block-index keys, RPC, and mempool all keep working), but the difficulty target is checked against the RandomX hash of the serialized header.

Why RandomX

  • CPU-friendly. Designed to run efficiently on commodity x86 CPUs. Anyone with a laptop can participate.
  • ASIC-resistant. Random program execution + 2 GB dataset means a custom chip would basically be a CPU.
  • 2048-block epoch rotation with 64-block lag. Seed for an epoch derives from the hash of a fixed past block, so miners can't precompute it indefinitely.

Mine via the pool

Easiest path: point a Pricoin-aware miner at pool.pricoin.org with your stealth address as the username. Payouts arrive directly as confidential outputs in your wallet — no registration or login. 

# Get a stealth address from your wallet:
pricoin-cli -rpcwallet=YOURWALLET pricoin_getstealthaddress
# → {"address": "H6..."}

# Mine with the Pricoin xmrig fork (rx/pric):
pricoin-miner \
    --algo=rx/pric \
    --url=stratum.pricoin.org:3333 \
    --user=H6...your-stealth-address \
    --pass=x \
    --keepalive

# Pool stats + worker lookup:
#   https://pool.pricoin.org/
#   https://pool.pricoin.org/worker/H6...your-stealth-address

Hardware-mode RandomX needs ~2 GB of RAM and benefits a lot from kernel hugepages — sudo sysctl -w vm.nr_hugepages=1280 is the recommended host-side tweak.

Solo mining still works the standard Bitcoin Core way (getblocktemplate + your own miner), but with any non-trivial network hashrate the pool is the practical path to consistent payouts.

Mainnet difficulty & ramp-up

The mainnet genesis is mined at the lowest possible RandomX difficulty target (0x207fffff). Post-genesis, Bitcoin's standard difficulty adjustment algorithm (DAA) kicks in: re-target every 2016 blocks (about 3.5 days at 150 s/block) so average inter-block time tracks the network hashrate.
